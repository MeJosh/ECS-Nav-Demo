# Support Local and Dedicated Simulation Hosts

The demo will support both a dedicated server Simulation Host and a single-user local Simulation Host while preserving one authoritative Simulation World in either mode. ECS world logic, systems, and command handling should live in a dedicated simulation package, shared boundary contracts should stay in the shared package, and the client should talk through a common session interface so remote and local modes use the same Input Command vocabulary. Local mode should be functionally identical to dedicated server mode from the client perspective, including command semantics, participant identity allocation, path editing behavior, snapshot shape, and simulation cadence, so it does not depend on special local-only participant IDs or a reduced feature set. Mode selection starts as build-time configuration, with an in-app picker and connectable ad-hoc hosting left for later decisions.
